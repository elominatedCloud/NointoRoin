# 65세 구직자 유저 여정 지도

## 페르소나

- 이름: 김영자, 65세
- 상황: 은퇴 후 월 50~80만원 정도의 보탬이 되는 짧은 일자리를 찾음
- 디지털 숙련도: 카카오톡과 전화는 익숙하지만, 작은 글자와 복잡한 메뉴는 부담스러움
- 건강 조건: 오래 서 있거나 무거운 물건을 드는 일은 피하고 싶음
- 신뢰 기준: 공공기관/주민센터/복지관처럼 확인 가능한 곳, 전화로 물어볼 수 있는 곳

## 여정

1. 불안 해소
   - "내가 해도 되는 일인가?", "사기 아닌가?", "어디에 전화해야 하나?"를 먼저 걱정함.
   - 첫 화면은 기능 목록보다 `우리동네 일자리`, `내 조건으로 추천`, `말로 도움받기` 세 가지 행동을 크게 보여준다.

2. 자기 조건 말하기
   - 긴 입력 폼 대신 한 번에 한 질문만 제시한다.
   - 질문 순서는 경력보다 건강 제한을 이른 단계에서 확인한다.
   - 선택지는 숫자와 짧은 문장으로 만들고 TTS로 다시 들을 수 있어야 한다.

3. 공고 이해
   - 공고 원문 제목은 유지하되 쉬운 설명, 지역, 기간, 몸에 무리가 될 수 있는 점을 먼저 보여준다.
   - 세부 조건이 없으면 "확인 필요"라고 명확히 말한다.

4. 행동 결정
   - 상세 화면 첫 화면 안에 `신청하기`가 보여야 한다.
   - 전화번호가 있으면 `전화하기`, 없으면 `전화번호 확인 필요`로 표시한다.
   - 신청 방법은 듣기 전용이 아니라 실제 다음 행동 순서로 보여준다.

5. 이후 관리
   - 이력서, 출근 달력, 마이 화면은 보조 기능이다.
   - 첫 홈의 주 행동을 방해하지 않되, 하단 내비게이션으로 언제든 돌아갈 수 있어야 한다.

## UI 원칙

- 한 화면에는 하나의 주 결정을 둔다.
- 버튼은 최소 64px 이상 높이와 큰 글자를 사용한다.
- 기능명보다 결과 중심 문구를 쓴다.
- 공공 API 데이터는 노인용 공고 화면에 우선 노출한다.
- 기업용 화면은 노인용과 다르게 조용하고 빠르게 스캔 가능한 관리 화면으로 유지한다.
