# 기업 담당자 유저 여정 지도

## 페르소나

- 이름: 이민수, 48세
- 역할: 복지관/마트/관리소의 채용 담당자 또는 현장 관리자
- 상황: 짧은 시간 일할 시니어 인력을 빠르게 찾고, 면담 후 바로 출근 일정을 잡아야 함
- 디지털 숙련도: 업무용 웹은 익숙하지만 시간이 부족함
- 신뢰 기준: 연락 가능 여부, 실제 근무 가능 시간, 건강상 주의점, 공고와의 적합도

## 여정

1. 처음 설정
   - 기업 정보와 연락처를 입력한다.
   - 공고에 노출될 위치, 업종, 담당자 연락처가 정확해야 한다.

2. 공고 등록
   - 하는 일, 근무 시간, 급여, 자격, 신청 방법을 입력한다.
   - 공고는 시니어가 이해하기 쉬운 설명으로 자동 변환되어야 한다.

3. 추천 이력서 검토
   - 적합도 높은 후보를 먼저 본다.
   - 담당자는 `왜 맞는지`, `주의할 점`, `전화 가능한지`를 빠르게 판단해야 한다.
   - 후보 카드에는 바로 전화하거나 면담 대상으로 표시하는 행동이 있어야 한다.

4. 면담과 채용 결정
   - 짧은 면담으로 가능 요일, 이동 거리, 건강상 제한, 서 있는 시간을 확인한다.
   - 실제 채용 여부는 데모에서는 상태 UI로만 표현한다.

5. 출퇴근 관리
   - 출근코드를 공유하고 월간 출근일, 예상 일급, 근속수당 조건을 확인한다.

6. 정책 확인
   - 시니어 채용 지원 제도, 면담 체크리스트, 주의사항을 필요할 때 확인한다.

## UI 원칙

- 기업용은 노인용보다 밀도가 높아도 되지만, 반복 업무를 빠르게 스캔할 수 있어야 한다.
- 홈은 `오늘 할 일`을 중심으로 구성한다.
- 후보 목록은 카드보다 표/행 중심으로 적합도, 연락, 주의점을 비교하기 쉽게 한다.
- 전화 가능한 후보는 `전화하기`를 실제 `tel:` 링크로 제공한다.
- 공공 일자리 API는 기업의 핵심 업무가 아니라 상태 지표/디버그 정보로만 둔다.
